# CLAUDE.md

## Overview

This repository **is** the Tomacheese Maven repository served at
<https://repo.tomacheese.com> (custom domain via `CNAME`, GitHub Pages). The
published Maven artifacts live at the repository root under `com/` and `dev/`;
they are deployed here by CI (`mvn deploy`), not authored by hand. `README.md`
is an auto-generated index of every artifact and its versions.

The only hand-maintained source code is the README generator under
`.github/generate-readme/` (TypeScript, run with pnpm on Node.js — see
`.node-version`).

## Repository layout

- `com/`, `dev/` — published Maven artifacts (`maven-metadata.xml`, jars, poms).
  Machine-generated by the deploy workflows; do not edit by hand.
- `README.md` — generated from all `maven-metadata.xml` files. Do not edit by
  hand; edit `.github/generate-readme/template.md` and/or the generator instead.
- `CNAME` — custom domain (`repo.tomacheese.com`). Do not remove.
- `.github/generate-readme/` — the README generator (see below).
- `.github/workflows/` — CI and deploy automation.

## Development commands

All commands run **inside `.github/generate-readme/`** (pnpm is enforced via
`preinstall` `only-allow pnpm`; do not use npm/yarn):

- `pnpm install --frozen-lockfile` — install dependencies.
- `pnpm start --target ../../ --output ../../README.md` — regenerate `README.md`
  by scanning the repo root for `maven-metadata.xml` files (this is exactly what
  the `generate-readme` workflow runs).
- `pnpm run lint` — Prettier check + ESLint + `tsc` (run in CI).
- `pnpm run fix` — auto-fix Prettier + ESLint.
- `pnpm run dev` — watch mode via tsx.

CI (`.github/workflows/ci.yml`) runs `pnpm run lint` and `npx depcheck` for the
generator on every push/PR to `main`/`master`.

## Generator internals (`src/main.ts`)

Single entry point. It recursively finds `maven-metadata.xml` files under
`--target`, parses each with `fast-xml-parser`, derives the latest version
(`versioning.latest` → `versioning.release` → `version`, else the file is
skipped with a warning), and substitutes the rendered list into
`template.md`'s `{{REPOSITORYS}}` placeholder, writing the result to `--output`.

## Coding conventions

- TypeScript is strict (`tsconfig.json`: `strict`, `noUnusedLocals`,
  `noUnusedParameters`, `noImplicitReturns`). Keep it compiling with no unused
  symbols.
- Prettier (`.prettierrc.yml`): no semicolons, single quotes, `printWidth` 80,
  trailing commas `es5`. Run `pnpm run fix` before committing.
- ESLint uses the shared `@book000/eslint-config`.

## Adding a new Maven project

Artifacts are added through the `workflow_dispatch` workflows
(`create-repo-maven.yml`, `jsa2-deploy.yml`), which run `mvn deploy` into this
repo and open a PR. Do not stage jars/poms under `com/` or `dev/` by hand.

## Prohibitions

- Do not hand-edit `README.md` or anything under `com/`/`dev/` — both are
  generated; changes will be overwritten by the next deploy/generate run.
- Do not commit secrets. The `generate-readme` workflow pushes with
  `PUSH_DEPLOY_KEY` supplied via GitHub Actions secrets, never checked in.
- Do not switch package managers away from pnpm.

## Documentation update rules

When you change the generator's behavior, its commands, or the workflow set,
update this file and (if the output format changes) `template.md` in the same
change.
